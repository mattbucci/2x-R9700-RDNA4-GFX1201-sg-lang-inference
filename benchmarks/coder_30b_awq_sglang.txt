Notes:
- Best throughput scaling: near-linear from 1→32 concurrency (29→332 tok/s)
- 3.3B active params per token = minimal memory bandwidth per request
- At high concurrency, multiple requests activate different expert subsets
  so total bandwidth grows sub-linearly while throughput scales ~11x
- Context scaling: stable ~27 tok/s up to 4K, drops at 8K+ as KV cache
  pressure limits batching
- Quality verified: math, code, knowledge tasks correct
- Three RDNA4-specific crash sources were fixed:
    1. sgl_kernel.topk_softmax → torch-native topk+softmax
    2. Triton AWQ GEMM → dequant+matmul for M>1, HIP GEMV for M=1
    3. Per-expert Python loop → HIP GEMV fused MoE dispatch
- vLLM Docker FP8 comparison: 94 tok/s single, 1,882 @64 (uses FP8 WMMA
  which is blocked on SGLang by Arch comgr bug)
